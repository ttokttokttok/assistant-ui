Use these APIs when your assistant needs to call application code, render a custom tool result, or expose tool state inside a React UI.

Tool **registration** has two scopes. Pick the one that matches the lifecycle you want for the tool. Tool **rendering** and **status** are independent of either choice and apply to both.

| | `Tools({ toolkit })` | `useAssistantTool` / `makeAssistantTool` |
| --- | --- | --- |
| Scope | Entire assistant subtree below the mount point | Only while the registering React component is mounted |
| Lifecycle | Tied to the runtime / provider tree | Tied to a specific UI surface |
| Typical use | App-wide tools, MCP merging, tools defined alongside the runtime | Tools whose availability depends on a UI being visible |
| Stable reference | Pass a stable `toolkit` (module scope or `useMemo`) | Pass a stable tool object (module scope or `useMemo`) |

Both styles can coexist in the same app, and tools registered through either path participate in rendering and status APIs the same way.
